## Overview

The Nexus Hotel platform uses Docker for containerization with separate configurations for development and production environments. The setup leverages multi-stage builds, Turborepo for monorepo management, and pnpm for efficient package management.

## Docker Files

### Production Dockerfile

The production Dockerfile (`Dockerfile`) uses a multi-stage build process to optimize image size and build efficiency:

```dockerfile
FROM node:22-slim AS base
ENV PNPM_HOME="/pnpm"
ENV PATH="$PNPM_HOME:$PATH"
RUN corepack enable

FROM base AS builder
WORKDIR /app
RUN npm install -g turbo
COPY . .
RUN turbo prune web --docker

FROM base AS installer
WORKDIR /app
COPY .gitignore .gitignore
COPY --from=builder /app/out/json/ .
COPY --from=builder /app/out/pnpm-lock.yaml ./pnpm-lock.yaml
RUN pnpm install

COPY --from=builder /app/out/full/ .
COPY turbo.json turbo.json
RUN npx turbo run build --filter=web...

FROM base AS runner
WORKDIR /app
COPY --from=installer /app .
EXPOSE 3000
CMD ["pnpm", "--filter", "web", "run", "start"]
```

**Build Stages:**

1. **base** - Sets up Node.js 22 with pnpm via corepack
2. **builder** - Installs Turbo and prunes the monorepo for the web app
3. **installer** - Installs dependencies and builds the project
4. **runner** - Final production image with built application

### Development Dockerfile

The development Dockerfile (`Dockerfile.dev`) provides a simpler setup for local development:

```dockerfile
FROM node:22-slim

WORKDIR /app

RUN npm install -g pnpm@latest turbo

COPY . .
RUN pnpm install

EXPOSE 3001
ENV NODE_ENV=development
ENV HOST=0.0.0.0
ENV PORT=3001

CMD ["pnpm", "turbo", "run", "dev", "--filter=web"]
```

<Note>
Development mode runs on port **3001** while production uses port **3000**.
</Note>

## Docker Compose

The `compose.yaml` file provides a simple orchestration for running the production build:

```yaml
services:
  api:
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - PORT=3000
      - HOST=0.0.0.0
    restart: always
```

## Building Images

<Tabs>
  <Tab title="Production">
    Build the optimized production image:
    
    ```bash
    docker build -t nexushotel-web:latest -f Dockerfile .
    ```
    
    The multi-stage build process will:
    - Prune the monorepo to only include the web app dependencies
    - Install only production dependencies
    - Build the application with Turbo
    - Create a minimal runtime image
  </Tab>
  
  <Tab title="Development">
    Build the development image:
    
    ```bash
    docker build -t nexushotel-web:dev -f Dockerfile.dev .
    ```
    
    This includes all development dependencies and tools for hot-reloading.
  </Tab>
</Tabs>

## Running Containers

<CodeGroup>
```bash Docker Run (Production)
docker run -p 3000:3000 \
  -e NODE_ENV=production \
  -e PORT=3000 \
  -e HOST=0.0.0.0 \
  nexushotel-web:latest
```

```bash Docker Run (Development)
docker run -p 3001:3001 \
  -e NODE_ENV=development \
  -e PORT=3001 \
  -e HOST=0.0.0.0 \
  -v $(pwd):/app \
  nexushotel-web:dev
```

```bash Docker Compose
docker compose up -d
```
</CodeGroup>

## Multi-Stage Build Benefits

The production Dockerfile uses a multi-stage build strategy that provides several advantages:

<CardGroup cols={2}>
  <Card title="Smaller Image Size" icon="compress">
    Only production dependencies and built artifacts are included in the final image
  </Card>
  
  <Card title="Build Caching" icon="clock">
    Dependencies are cached separately from source code for faster rebuilds
  </Card>
  
  <Card title="Security" icon="shield">
    Build tools and source code are not present in the final runtime image
  </Card>
  
  <Card title="Monorepo Optimization" icon="layer-group">
    Turbo prune extracts only required workspace dependencies
  </Card>
</CardGroup>

## Best Practices

<Warning>
**Security Considerations:**
- Never commit `.env` files with secrets to version control
- Use Docker secrets or environment variable injection for sensitive data
- Regularly update the base Node.js image for security patches
- Scan images for vulnerabilities using tools like `docker scan`
</Warning>

<Tip>
**Optimization Tips:**
- Use `.dockerignore` to exclude unnecessary files from the build context
- Leverage Docker layer caching by copying package files before source code
- Use `--filter=web...` in Turbo builds to include only necessary dependencies
- Consider using `pnpm fetch` for better caching in CI/CD pipelines
</Tip>

## Troubleshooting

### Build Failures

If the build fails during the `pnpm install` step:

```bash
# Clear Docker build cache
docker builder prune -a

# Rebuild without cache
docker build --no-cache -t nexushotel-web:latest .
```

### Port Conflicts

If ports 3000 or 3001 are already in use:

```bash
# Find process using the port
lsof -i :3000

# Run on different port
docker run -p 3002:3000 nexushotel-web:latest
```
