## Overview

Nexus Hotel is built using modern DevOps practices with a focus on security, availability, and continuous deployment. The platform leverages a comprehensive technology stack across frontend, backend, infrastructure, and operations.

## Containerization

### Docker

<ParamField path="Container Platform" type="Docker">

</ParamField>

**Images:**

#### Production Image
- **Base:** `node:22-slim`
- **Build Strategy:** Multi-stage
- **Registry:** Docker Hub
- **Repository:** `ssubaru/nexushotel`
- **Tag:** `latest`

**Build Stages:**
1. **base** - pnpm setup
2. **builder** - Turbo workspace pruning
3. **installer** - Dependency installation
4. **runner** - Minimal production runtime

- **Base:** `node:22-slim`
- **Features:** Hot-reload, dev tools
- **Port:** `3001`

## Technology Decision Rationale

### Why Next.js?

- Server-side rendering for SEO
- Excellent developer experience
- Built-in optimizations
- Vercel deployment integration

### Why Kubernetes?

- High availability (3 replicas)
- Auto-scaling capabilities
- Industry-standard orchestration
- Declarative configuration

### Why Terraform?

- Infrastructure versioning
- Automated provisioning
- Multi-cloud support
- State management

### Why Supabase?

- Open-source
- PostgreSQL reliability
- Built-in authentication
- Real-time capabilities
- Row-level security
