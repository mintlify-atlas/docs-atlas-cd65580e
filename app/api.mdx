## API Implementation

The main API file (`apps/api/src/index.ts`) is minimal:

```typescript apps/api/src/index.ts
import { CONSTANT } from '@repo/constants'

export default {
  fetch: () => new Response(CONSTANT)
}
```

<Note>
  This simple implementation follows the Web Standard `fetch` API pattern, making it compatible with various JavaScript runtimes (Bun, Deno, Cloudflare Workers, Node.js).
</Note>

## Key Features

### Runtime Agnostic

The API uses the standard `fetch` handler interface:

```typescript
export default {
  fetch: (request: Request) => Response | Promise<Response>
}
```

This makes it compatible with:
- Bun
- Deno
- Cloudflare Workers
- Node.js (with adapter)

### Shared Constants

Imports from the monorepo's shared constants package:

```typescript
import { CONSTANT } from '@repo/constants'
```

**Constant value** (from `packages/constants/src/constants.ts`):

```typescript packages/constants/src/constants.ts
export const CONSTANT = 'Hello from your Hono API!'
```

## Development

### Running the API

<Tabs>
  <Tab title="Development">
    Start the development server with hot reload:
    
    ```bash
    pnpm dev
    ```
    
    This runs `srvx` which watches for file changes.
  </Tab>
  
  <Tab title="Build">
    Compile TypeScript to JavaScript:
    
    ```bash
    pnpm build
    ```
    
    Output goes to the `dist/` directory.
  </Tab>
  
  <Tab title="Production">
    Start the production server:
    
    ```bash
    pnpm start
    ```
    
    Runs the compiled code from `dist/index.js`.
  </Tab>
</Tabs>

### Development Workflow

<Steps>
  <Step title="Install Dependencies">
    ```bash
    pnpm install
    ```
  </Step>
  
  <Step title="Start Dev Server">
    ```bash
    cd apps/api
    pnpm dev
    ```
    
    Or from the root:
    ```bash
    pnpm dev
    ```
  </Step>
  
  <Step title="Make Changes">
    Edit `src/index.ts` and the server will automatically reload.
  </Step>
  
  <Step title="Build for Production">
    ```bash
    pnpm build
    ```
  </Step>
</Steps>

## Extending the API

To add more endpoints, you can expand the API using Hono's routing:

<Tabs>
  <Tab title="Basic Route">
    ```typescript
    import { Hono } from 'hono'
    import { CONSTANT } from '@repo/constants'
    
    const app = new Hono()
    
    app.get('/', (c) => c.text(CONSTANT))
    app.get('/health', (c) => c.json({ status: 'ok' }))
    
    export default app
    ```
  </Tab>
  
  <Tab title="With Params">
    ```typescript
    import { Hono } from 'hono'
    
    const app = new Hono()
    
    app.get('/rooms/:id', (c) => {
      const id = c.req.param('id')
      return c.json({ roomId: id })
    })
    
    export default app
    ```
  </Tab>
  
  <Tab title="POST Endpoint">
    ```typescript
    import { Hono } from 'hono'
    
    const app = new Hono()
    
    app.post('/bookings', async (c) => {
      const body = await c.req.json()
      // Process booking
      return c.json({ success: true, booking: body })
    })
    
    export default app
    ```
  </Tab>
</Tabs>

## Integration with Frontend

The API can be called from the Next.js frontend:

```typescript
// In a React component or API route

const data = await response.text()
console.log(data) // "Hello from your Hono API!"
```

## srvx Development Server

`srvx` is a lightweight development server that:
- Watches for file changes
- Automatically recompiles TypeScript
- Restarts the server on changes
- Provides fast feedback loop
