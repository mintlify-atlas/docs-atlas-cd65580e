## Overview

Get the Nexus Hotel reservation platform up and running on your local machine quickly. This guide will walk you through cloning the repository, installing dependencies, and starting the development server.

<Note>
  This is a monorepo project using **Turborepo** and **pnpm** workspaces with Next.js for the frontend and a Node.js API.
</Note>

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** 22.x or later
- **pnpm** 10.28.2 or later (recommended)
- **Git** for version control

<Tip>
  The project uses `pnpm@10.28.2` as specified in the `packageManager` field. It will be automatically used if you have Corepack enabled.
</Tip>

## Quick Start Steps

<Steps>
  <Step title="Clone the Repository">
    Clone the Nexus Hotel project from your Git repository:

    ```bash
    git clone <repository-url>
    cd proyecto-de-devops
    ```
  </Step>

  <Step title="Install Dependencies">
    Install all workspace dependencies using your preferred package manager:

    <CodeGroup>
      ```bash pnpm
      pnpm install
      ```

      ```bash npm
      npm install
      ```

      ```bash yarn
      yarn install
      ```
    </CodeGroup>

    <Note>
      The project is optimized for **pnpm**. Using npm or yarn may work but is not officially supported.
    </Note>
  </Step>

  <Step title="Configure Environment Variables">
    Create a `.env` file in the root directory by copying the example:

    ```bash
    cp .env.example .env
    ```

    Update the `.env` file with your configuration:

    ```bash .env
    PORT=3001
    DATABASE_URL=your-supabase-database-url
    API_KEY=your-api-key
    ```

    <Warning>
      You'll need a Supabase project to get the `DATABASE_URL`. See the [Environment Setup](/environment-setup) guide for details.
    </Warning>
  </Step>

  <Step title="Start the Development Server">
    Run the development server using Turborepo:

    <CodeGroup>
      ```bash pnpm
      pnpm dev
      ```

      ```bash npm
      npm run dev
      ```

      ```bash yarn
      yarn dev
      ```
    </CodeGroup>

    This command starts all applications in the monorepo in watch mode:
    - **Web app** (Next.js): http://localhost:3001
    - **API**: Runs on configured port
  </Step>

  <Step title="Verify Installation">
    Open your browser and navigate to:

    ```
    http://localhost:3001
    ```

    You should see the Nexus Hotel reservation platform homepage.
  </Step>
</Steps>

## Project Structure

The monorepo is organized as follows:

```
proyecto-de-devops/
├── apps/
│   ├── web/          # Next.js frontend application (port 3001)
│   └── api/          # Node.js API backend
├── packages/
│   ├── ui/           # Shared UI components
│   ├── constants/    # Shared constants
│   └── config/       # Shared configurations
├── turbo.json        # Turborepo configuration
├── pnpm-workspace.yaml
└── package.json      # Root workspace configuration
```

## Available Scripts

From the root directory, you can run:

| Command | Description |
|---------|-------------|
| `pnpm dev` | Start development servers for all apps |
| `pnpm build` | Build all applications for production |
| `pnpm lint` | Run ESLint across all workspaces |
| `pnpm format` | Format code using Prettier |
| `pnpm check-types` | Run TypeScript type checking |

## Common Issues

<AccordionGroup>
  <Accordion title="Port 3001 is already in use">
    If port 3001 is occupied, you can change it in `apps/web/package.json`:

    ```json
    "scripts": {
      "dev": "next dev --turbopack --port 3002"
    }
    ```
  </Accordion>

  <Accordion title="pnpm command not found">
    Install pnpm globally:

    ```bash
    npm install -g pnpm@10.28.2
    ```

    Or enable Corepack:

    ```bash
    corepack enable
    ```
  </Accordion>

  <Accordion title="Database connection errors">
    Ensure your `DATABASE_URL` in `.env` is correct and your Supabase project is active. See [Environment Setup](/environment-setup) for configuration details.
  </Accordion>
</AccordionGroup>
